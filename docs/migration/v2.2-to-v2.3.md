# Migration Guide: v2.2 → v2.3

## Overview

v2.3.0 restores `prp-review-agents` from a 9-line alias to a standalone 1141-line command that spawns **actual parallel Agent subprocesses** via the Agent tool. **No breaking changes** — `run-all`, all flags, and artifact formats work identically.

## What Changed

### 1. `prp-review-agents` is no longer an alias

**Before (v2.2)**: `prp-review-agents` → alias → `prp-review` (same single-session 11-pass review)
**After (v2.3)**: `prp-review-agents` → spawns 3+ parallel Agent subprocesses → synthesizes → posts to GitHub

**Action required**: None. The command name, arguments, and output format are unchanged.

### 2. New canonical prompt

**New file**: `prompts/review-agents.md` (1141 lines)

This is now a standalone prompt (not an alias) in the `prompts/` directory. The generator reads it directly and produces full adapter files for all 5 tools.

**For contributors**: If you edit `prompts/review-agents.md`, regenerate adapters as usual:

```bash
python3 scripts/generate-adapters.py
```

### 3. Agent set changes

| Agent | v2.2 | v2.3 |
|-------|------|------|
| `silent-failure-hunter` | Conditional | **Always-run (core)** |
| `dependency-analyzer` | Always-run | **Conditional** (package files changed) |
| `docs-impact-agent` | "Almost always" | Conditional (production code changed) |

### 4. New error handling

v2.3 adds handling for scenarios that were previously silent:

- Core agent failure → forces NEEDS FIXES verdict minimum
- `--context` path validation (rejects traversal, validates non-empty diff)
- Context file creation verified before agent dispatch
- No-PR-found → explicit STOP with diagnostic message
- `gh pr review` permission failure → logs error, falls back to comment
- Metrics append failure → WARNING (non-fatal)

### 5. New metrics field

Review metrics JSONL now includes `"review_type": "agents"` to distinguish from single-session reviews (`"single"`).

## How to get old behavior

If you prefer the single-session sequential review:

```bash
# Use prp-review directly
/prp-core:prp-review 42

# Or in run-all, use --review-single-agent flag
/prp-core:prp-run-all "feature" --review-single-agent
```

## Update PRP in your projects

```bash
cd .prp && git pull origin main
./scripts/install.sh "$(pwd)/.."
```

Or globally:

```bash
prp-install-all
```
